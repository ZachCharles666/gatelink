# GateLink x Hyperliquid PRD v0.3

## 1. 产品概述

GateLink 是一个基于 Hyperliquid 资金体系的 AI API 交易平台。

买家通过钱包登录平台，充值 USDC 获得 Credits，再使用 GateLink API Key 调用平台能力并消耗 Credits。卖家通过钱包登录，绑定收款地址、上架资源、获取收益，并由平台统一结算。

## 2. 版本范围

`v0.3` 包含：
- 买家端 Web
- 卖家端 Web
- 管理端 Web
- 平台后台服务文档
- Hyperliquid 资金流与平台账本文档

`v0.3` 不包含：
- 邮箱/手机号注册登录
- 自研托管钱包
- 每次 API 请求都上链结算
- 将卖家原始 Key 暴露给买家

## 3. 用户角色

- 买家：充值 Credits、生成 API Key、消费平台能力、查看账单、发起退款
- 卖家：绑定收款地址、配置资源、获得收益、申请结算
- 管理员：处理充值、退款、卖家结算、风控与对账

## 4. 核心业务规则

### 4.1 钱包登录

- v0.3 仅支持钱包签名注册/登录
- 首次签名创建账号，再次签名完成登录
- 默认一个钱包绑定一个平台主体

### 4.2 Credits 制度

- 买家充值 USDC 后获得 Credits
- 建议前台统一展示为 Credits，后台按 USD/USDC 账务处理
- 默认兑换关系：`1 USDC = 1 Credit`

### 4.3 API 消耗

- 买家使用 GateLink API Key 调用平台 API
- 平台在请求完成后按实际 usage 扣减 Credits
- 流式请求必须等流结束后再记账

### 4.4 平台抽佣

- 平台不在充值时抽佣
- 平台在每次 usage 完成后按实际消耗拆账
- 平台佣金 = 买家扣费 - 卖家应得

### 4.5 链上与账本分层

- 链上处理：充值、退款、卖家结算
- 平台账本处理：Credits 扣减、usage 记账、卖家待结算、平台佣金

## 5. Hyperliquid 资金原则

- Hyperliquid 只承担大额入金和出金
- 不对每次微小 API 消耗上链
- 平台可使用主资金账户 + agent wallet + 冷备钱包架构
- `engine/`、`api/`、`web/` 仍保留现有系统分层，不因文档版本升级而移动目录

## 6. 成功标准

- 买家能用钱包登录并完成充值获得 Credits
- 买家能创建并使用平台 API Key
- 卖家能绑定收款地址并完成结算
- 管理员能查看充值、退款、结算、风险与对账
- 当前文档结构可支撑后续 `v0.4+` 继续演进
