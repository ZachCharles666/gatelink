# UI Guidelines v0.3

## 1. 设计方向

- 参考 OpenAI 的克制感和清晰层级
- 参考 OpenRouter 的开发者控制台信息组织
- 不做视觉复制

## 2. 视觉关键词

- 简洁
- 专业
- 稳定
- 数据透明
- 开发者友好

## 3. 布局规范

- 桌面端：左侧导航 + 顶部状态栏 + 主工作区
- 移动端：顶部页头 + 抽屉或底部导航

## 4. 组件规范

### 指标卡

- 标题
- 主数值
- 状态
- 辅助说明

### 钱包卡片

- 地址
- 连接状态
- 网络
- 复制
- 浏览器跳转

### 交易表格

- 状态
- 金额
- 时间
- 对手方
- tx hash
- 操作

## 5. 状态色

- `pending`：蓝色
- `success / confirmed / paid`：绿色
- `warning / review`：黄色
- `failed / rejected / suspended`：红色

## 6. 文案规范

- 面向买家的余额统一叫 `Credits`
- 链上相关动作必须提示不可逆或到账确认规则
- 不出现“你已获得卖家原始 Key”之类文案
