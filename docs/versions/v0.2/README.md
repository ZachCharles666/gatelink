## 本目录用途

本目录用于承载 **当前版本 v0.2** 的产品功能逻辑与任务拆分，不再沿用早期 MVP 启动期的分工式文档组织方式。

## 目录结构

- `buyer/FEATURES.md`
  - 买家端功能逻辑、用户流程、页面职责
- `buyer/PHASES.md`
  - 买家端当前版本的阶段任务拆分
- `seller/FEATURES.md`
  - 卖家端功能逻辑、用户流程、页面职责
- `seller/PHASES.md`
  - 卖家端当前版本的阶段任务拆分
## 使用方式

1. 根目录 [STATUS.md](/Users/tvwoo/Projects/gatelink/STATUS.md) 负责团队当前状态与进度
2. 根目录 [WORKFLOW.md](/Users/tvwoo/Projects/gatelink/WORKFLOW.md) 与 [RELEASES.md](/Users/tvwoo/Projects/gatelink/RELEASES.md) 负责仓库级协作与发布规则
3. 本目录负责说明“当前版本到底要做什么”
4. 以后若进入新版本，新增 `docs/versions/vX.Y/`，不要直接覆盖本目录

补充说明：

- 根目录协作与发布规则是跨版本通用的
- `docs/versions/v0.2/` 只负责 `v0.2` 的产品功能与任务，不承载仓库级工作流
